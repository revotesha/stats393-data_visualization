Introduction

In this tutorial, we will create a (better) visualization for a given data set which is a record of tornadoes that have happened in the United States between 1950 and 2015.

Original data

The original visualization of part of this data is two pie charts; one shows the distribution of the tornadoes by scale and the other shows the distribution of tornado-related deaths, also by tornado scale. The pie charts were used by ‘Tornado Project Online’ to show that, although tornadoes of scales 4 and 5 are rare, they cause more deaths than all the other tornadoes combined.

See pie charts

The tornado scale used here is the Enhanced Fujita (EF) scale. The EF scale is based on the damage the tornado causes and it ranges from 0 to 5, with 5 being the most violent tornadoes. According to the National Weather Service of the United States, the scale of a tornado is determined by first identifying the appropriate damage indicators from a list of 28. For the given indicator, the degree of damage is determined from a list of 8. Each degree of damage is assigned a range of wind speed, and it is from these ranges that the scale is finally decided.

The reason I decided to create a better way to visualize the distribution of tornadoes and tornado-ralated deaths is because (1) I find it unnessary to use 3-D (AND exploded) pie charts and I think a regular one-dimensional piechart is better and, (2) in general, pie charts can be confusing to interpret as they don’t always present the data in an easily digestible way - it can be hard to keep track of the labels and see the relationship of the sizes of the slices across groups (the several piecharts being compared).

In this tutorial we will find a better way to visualize the data to show the intended relationship. We will be working primarily with tidyverse, a collection of packages that work very well with tidy data. This package and a few others have been loaded for you, so you will not have to worry about it.

Putting it together

We will now put the two plots together to have one plot object. The plot object is diverging barplots. To get there, we will need to flip the coordinate system for both graphs and reverse the direction bars on one, so that the bars for each plot are pointing in the opposite directions. We will call the plot pointing to the right right_plot and the other left_plot. The two plots will share the same x-axis (which will be vertical after flipping the coordinates), so one will not have x-axis label and ticks.

Below is the code for this. Change the theme to theme_bw and remove the grid lines that appear on the plots. After that, change the color to something else for each plot. While trying to remove grid lines, know that a plot may have both major and minor grid lines and you need to account for both in your code. You will need to use both panel.grid.major and panel.grid.minor. Set these to element_blank().

Visualizaton: Piecharts

If you still prefer pie charts, here is a better way to use piecharts with this data.

First, we make a data frame containing a column with mean death for each EF scale. Try do this. Hint: we did this when creating a barplot for fatalities.

mean_deaths = tornadoes %>%
  group_by(EF) %>%
  summarize(mean_deaths_per_sc = mean(fat)) %>%
  mutate(EF = as.factor(EF)) # we need to change EF to type factor to discretize
# coloring later on.

We then create another data frame with the number of tornadoes for each scale. This should be somewhat similar to creating expectation of number of fatalities for each scale. See if you can figure out the code. We will later need to bind the two data frames, so we can’t have repeating columns. Use a pipe operator and select(-EF) to remove the column.

n_tornadoes = tornadoes %>%
  group_by(EF) %>%
  summarize(num_tornadoes_per_sc = n()) %>%
  select(-EF)

In the code below, we do a column bind on the two data frames. We then change the resulting data frame to obtain a long data format. We will gather means and number of tornadoes for each scale into one column and have a corresponding column that helps us (or rather, the computer) what those values came from. We will call the column with values values and the other quantity. The columns mean_deaths_per_sc and num_tornadoes_per_sc are coming from mean_deaths data frame and n_tornadoes respectively. We then use a mutate function to change mean_deaths_per_sc to Average Deaths by Scale and num_tornadoes_per_sc to Tornado Count by Scale. Why do you think we are doing the last step? Hint: look at the titles for the pie charts.

both = cbind(mean_deaths, n_tornadoes) %>%
  gather(quantity, values, mean_deaths_per_sc, num_tornadoes_per_sc) %>%
  mutate(quantity = ifelse(quantity == "mean_deaths_per_sc",
                           "Average Deaths by Scale", 
                           "Tornado Count by Scale"))
# see what the data frame from the previous code looks like.

Below is the code the generates the pie charts. Read through it and try to understand what the different parts do. scale_fill_viridis is a color palette function – we have five categories we want to color and this picks the colors for us. Inside this function, we have set discrete to TRUE to discretize the colors. If you changed this to FALSE, you would get an error because we previously changed EF to be of type factor.

ggplot(data = both, aes(x = 1, y = values, fill = EF)) +
  geom_bar(stat = "identity", position = "fill") +
  coord_polar(theta = "y") +
  facet_wrap(~quantity) +
  theme_bw() +
  scale_fill_viridis(discrete = TRUE) + # could also have used
  # scale_fill_viridis_d(). d is for discretized values.
  theme(axis.title = element_blank(),
        axis.text = element_blank(),
        axis.ticks = element_blank(),
        panel.grid.major = element_blank(),
        panel.grid.minor = element_blank(),
        panel.border = element_blank())

Unlike on the website, these piecharts are drawn side by side, have colors that match and they share the same legendand. It is easier to compare this way.

Conclusion

Compared to the original charts, both results make it easier to compare the number of tornadoes by scale with the number of deaths for each scale. In my opinion, the barplots make this even easier as the bars are paired and are back to back. One can clearly see that, tornado count is highest for scale EF0 tornadoes, but the corresponding bar for deaths per EF scale is negligibly small. The reverse is true for EF5.
